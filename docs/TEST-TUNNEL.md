# Tunnel App Test Plan

## 概述

Tunnel App 是 ClawParty 的零信任隧道应用，消除物理距离限制，允许从任何位置访问设备。本文档记录了 Tunnel App 的测试方案和测试案例。

## 测试范围

### 1. 基础功能测试
- CLI 帮助命令
- 隧道类型支持（TCP/UDP）
- 隧道端点类型（inbound/outbound）

### 2. TCP 隧道测试
- 创建出站端点
- 创建入站端点
- 隧道连接
- 负载均衡

### 3. UDP 隧道测试
- 创建 UDP 隧道
- UDP 数据传输

### 4. 隧道管理测试
- 列出隧道
- 查看隧道详情
- 删除隧道

### 5. 多目标测试
- 单目标隧道
- 多目标负载均衡

### 6. 网络环境测试
- 跨网络连接
- 防火墙穿透

---

## 测试案例

### TC-TN-001: CLI 帮助命令

**前置条件**: Tunnel App 已启动

| 步骤 | 操作 | 预期结果 |
|------|------|----------|
| 1 | 执行 `ztm tunnel help` | 显示帮助信息 |
| 2 | 执行 `ztm tunnel get --help` | 显示 get 命令帮助 |
| 3 | 执行 `ztm tunnel open --help` | 显示 open 命令帮助 |
| 4 | 执行 `ztm tunnel close --help` | 显示 close 命令帮助 |

---

### TC-TN-002: 创建 TCP 出站端点

**前置条件**: 存在可访问的目标服务

| 步骤 | 操作 | 预期结果 |
|------|------|----------|
| 1 | 执行 `ztm tunnel open outbound tcp/greeting --targets 192.168.1.100:8080` | 出站端点创建成功 |
| 2 | 执行 `ztm tunnel get outbound` | 显示创建的端点 |
| 3 | 验证端点信息 | 包含名称和目标地址 |
| 4 | 重复创建同名端点 | 返回错误或覆盖 |

---

### TC-TN-003: 创建 TCP 入站端点

| 步骤 | 操作 | 预期结果 |

## 已知问题和注意事项

1. **端口范围**: 入站端点需要可用的本地端口
2. **防火墙**: 需要确保端点间可以通信
3. **负载均衡**: 多目标使用轮询算法
4. **协议支持**: 支持 TCP 和 UDP
5. **连接超时**: 默认超时设置需要根据网络环境调整
